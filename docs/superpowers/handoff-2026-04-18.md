## Key Decisions Locked In

- Backend: Java + Quarkus modular monolith
- Frontend: Next.js + React + TypeScript
- Database: PostgreSQL
- Graph UI: React Flow
- Content assets stay on disk and are loaded only when a node is opened
- Postgres stores roadmap, topic, edge, layout, and asset-reference metadata
- Shared topics across roadmaps
- Relationships are roadmap-specific
- Content is roadmap-specific
- Flat graph for MVP, no nested topic tree
- Catalog page plus direct roadmap/topic URLs
- Topic detail pages should be shareable and server-rendered
- Node positions are stored in DB; fallback auto-layout is allowed when positions are missing
- No auth, progress tracking, admin UI, or visual editor in the MVP
